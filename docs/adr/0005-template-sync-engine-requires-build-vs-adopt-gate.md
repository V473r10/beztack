# Template sync engine requires a build-vs-adopt gate

Template sync must not be built around the existing WIP CLI by inertia: that branch is a useful prototype, but not the Template source contract. Before landing a sync engine, Beztack will run a build-vs-adopt spike against battle-tested template/update tools such as Copier, Cruft/Cookiecutter, Git subtree, and Copybara using both a minimal versioned fixture and a real Beztack-to-lncd validation case; the selected design must support non-interactive agent/CI operation, PR-only application to Derived projects, Template revision as the primary Origin baseline, versionable Sync policy, reviewable conflicts, Trusted versus Community Derived project trust boundaries, and opt-in Promotion from reviewed PRs. The spike must produce a decision matrix with reproducible checks and a single recommendation to adopt, wrap, or build custom; Beztack may still keep custom code for Sync policy, Origin baseline compatibility, Sync state, and Promotion, but only where existing tools fail the accepted criteria.
