# Subscription projection owns provider event projection

The Subscription projection Module will be the deep Seam for turning provider events into local Subscription and Membership state. `/api/subscriptions/webhooks` is the canonical webhook URL, while `/api/payments/mercado-pago/webhook` remains a compatibility Adapter; route Adapters validate transport and pass a provider event envelope to the Module.

Provider state is authoritative and local state is a cache, so the projection Implementation fetches current provider resources through a narrow provider Adapter, projects current provider state rather than applying event transitions, owns `webhookLog`, and uses a durable event key for idempotency. The event key uses provider plus provider event or delivery id when available, falling back to provider, event type/action, resource id, and a stable raw-payload hash. The first slice includes `payment.approved` because prorated upgrade/downgrade correctness depends on Payment approval ordering; provider follow-up writes such as adjusting the new Subscription amount and cancelling the previous Subscription stay inside the Subscription projection Implementation for Locality.
