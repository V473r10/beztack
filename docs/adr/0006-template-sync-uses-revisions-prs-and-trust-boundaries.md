# Template sync uses revisions, PRs, and trust boundaries

Beztack template sync is modeled around explicit Template revisions and PR review rather than direct mutation of Derived project main branches. A Derived project records the Template revision it last accepted as its Origin baseline, receives Template updates through branch/PR application, and uses Sync policy to preserve Product domain ownership; Promotions flow back to Beztack only from opt-in reviewed PRs and are filtered by ownership, while Product domain ideas require Platform extraction instead of direct copying. Trusted Derived projects are identified by a Beztack-owned registry and may use trusted automation, but Community Derived projects are treated as untrusted external contributors whose Promotions enter as normal PRs or patches validated by Beztack CI.
