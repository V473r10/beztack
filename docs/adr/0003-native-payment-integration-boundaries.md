# Payment integrations use native provider boundaries

Beztack isolates Subscriptions by each Payment provider's native application or credential boundary instead of adding a Beztack-owned integration namespace to provider metadata. Mercado Pago uses its Application identity as the authoritative boundary and must fail closed when that identity is missing or mismatched; the Mercado Pago integrator ID is only partner tracking and is not an access boundary. Polar remains isolated by its existing organization/token scope unless future evidence shows otherwise. We rejected a Beztack-owned namespace because it would add cross-provider metadata complexity and would not reinforce the provider-native boundary that already determines which resources an integration is allowed to create and manage.
